The refinement rounds

How the winner got sharp

Prefix-clustering won the bake-off at 85% precision. Then three rounds — each one a real bug hunted down on real titles. Precision stayed above 96% the whole way while recall climbed from 44% to 61%.

The junk-reduction loop · 2 rounds

Chasing junk below 0.5%

A bounded adjudication loop (max 5 rounds, exited at 2). Each round: find the winner's junk arcs, have two blind Sonnet judges + a reconciler rule each one REAL (a real arc the labelers missed → fix the gold set) or JUNK (→ a structural detector guard). Precision and recall were protected the whole way — a guard was kept only if it didn't cost recall.

Irreducible floor

Diss and Tell — a recurring segment. Structurally identical to a real 2-part arc, so regex can't remove it without killing real ones. This is the floor.

The same one arc is the floor for both junk and precision

A guard that dropped “big pipe number” clusters would have killed 18 real arcs in Even the Rich, which numbers every arc by episode number (Taylor Swift: Fearless | … | 212). A two-part story and a two-entry recurring segment are the same title shape — separating them needs meaning, not pattern. That single unremovable arc (Diss and Tell, 2 episodes) caps junk at 0.28% and caps membership precision at 99.84% on this gold set. Going lower on either would require reading episode descriptions with an LLM, which the on-device, no-LLM constraint rules out — so both loops stopped, honestly, at one irreducible arc.

The last 0.9% was duplicate episodes

Of 1,227 episodes placed into arcs, only 13 were misplaced — and 11 were the same arc re-aired. Feeds that re-run a series (… Redux, ENCORE: …, (Archive Episode)) made the detector emit one bloated arc with two of every part.

11

Re-release duplicates, collapsed

Lions, Coyotes & Bears Parts 1–4 Redux; Christina Pipkin archive re-airing; two ENCORE Gwyneth Paltrow parts. A structural dedup drops a re-release only when a non-re-release sibling covers that part — never a distinct episode.

The winning recipe

All ICU-compatible, so it drops into NSRegularExpression. Reference implementation lives in curation/arc-bakeoff/approaches.py.

# 1 · clean the title
strip noise prefix  Encore|Fan Favorite|Announcing|Update|Bonus|Replay|Revisited|…
strip episode id    \s*\|\s*#\d+\s*$
strip trailing []   only when it holds no counter — keep "(Part 1)", drop "[Repetición]"

# 2 · extract (stem, part) — first match wins; part = digits | roman | one..twenty
pipe        ^(.+?)\s*\|\s*.+?\s*\|\s*(\d+)$          # American History Tellers
part/paren  - Part N   , Part N   (Part N)   (Pt N)       # Explorers, The Constant
hash / ep   #N     - Ep N     Chapter N                 # 99% Invisible, Serial
general     ^(.+?)[\s:,.|]+Part[e]?\s+()\b   # dashes FIRST = literal!
general     Vol(ume) N
reject stem ^(season|episode|ep|part|chapter|vol|series|book|no)\b  or  < 3 chars

# 3 · cluster
group by normalized stem → prefix-merge (fold long stem into shorter prefix)
             → split a cluster across distinct itunes:season → keep groups of ≥ 2

# 4 · anthology guard (recall-safe)
drop cluster if counter kind is Volume and its smallest N ≥ 3   # open anthology, not a bounded arc
# do NOT drop on big pipe/episode numbers — Even the Rich numbers real arcs that way

# 5 · re-release dedup (recall-safe)
drop a Encore|Redux|Archive|Rebroadcast|Replay|… episode # only if a non-re-release sibling
             # already covers that part number — never drops a distinct episode

Known limits — kept honest

  • Untitled season arcs (e.g. Suave, where only itunes:season ties a season into one story) aren't caught — the season-grouping that would find them measurably hurts precision, so it's out on purpose.
  • Anthology series that look numbered but aren't narratives (Mini-Stories: Volume N) are the main residual junk.
  • Recall is a floor, not a ceiling. 60.7% at a precision-first setting; loosening the counter requirement trades precision below the baseline-beating bar.
